# Security Policy

Jan Suraksha is a digital crime reporting portal aimed at enabling citizens to report incidents securely. Since the platform may handle sensitive personal information, we take security and privacy seriously.

---

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

⚠️ **Do not create a public GitHub issue for security vulnerabilities.**  
Public disclosure may expose users or sensitive system behavior.

### How to Report
Please report security issues privately by contacting the project maintainers through:
- GitHub direct message to the repository owner/maintainers, or
- Any official contact method provided in the repository (if available)

When reporting, include:
- A clear description of the vulnerability
- Steps to reproduce the issue (if possible)
- Expected vs actual behavior
- Screenshots/logs (only if safe to share)
- Potential impact (data exposure, unauthorized access, etc.)

---

## Scope

Security-related issues may include (but are not limited to):
- Authentication and authorization flaws
- Data leakage or privacy issues
- Unsafe file uploads
- Injection vulnerabilities (SQLi, XSS, etc.)
- Broken access control
- Insecure communication or storage of complaint/user details

---

## Responsible Disclosure

Please allow maintainers reasonable time to investigate and fix the issue before publicly disclosing it.

We appreciate your efforts in helping keep Jan Suraksha secure and trustworthy.
